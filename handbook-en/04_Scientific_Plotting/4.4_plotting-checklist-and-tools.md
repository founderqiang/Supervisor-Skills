# 4.4 Plotting Checklist and Tool Quick Reference

## Plotting Checklist: Pre-submission Self-check List

Before inserting figures into the paper, please go through the following checklist one by one:

| Check Item | Passed |
|--------|---------|
| Are the figures exported in vector format (PDF/EPS/SVG)? | ☐ |
| Are the fonts in the figures clearly readable after LaTeX compilation (no smaller than 8pt)? | ☐ |
| Is the caption complete, allowing readers to understand the figure's content independently? | ☐ |
| Are the axis labels complete, including units? | ☐ |
| Is the legend clear, with a one-to-one correspondence to the elements in the figure? | ☐ |
| Have you used dual encoding with both color + line style/marker (colorblind friendly)? | ☐ |
| Are the Y-axis origin and range reasonable, without misleading the reader? | ☐ |
| Are the module names/symbols in the figure consistent with the body text? | ☐ |
| Is your method (Ours) visually appropriately highlighted? | ☐ |
| Are the figures generated by code (reproducibility)? | ☐ |

## Paper Plotting Tool Quick Reference

Finally, a summary of the recommended tool choices for different types of figures:

| Chart Type | Preferred Tool | Alternative Tool | Notes |
|---------|---------|---------|------|
| Motivated Example Figure | PPT / Figma / OmniGraffle | draw.io | Contains mixed elements such as text, icons, and code snippets, requires flexible layout |
| Solution Overview Figure | draw.io / PPT | Figma / TikZ | Flowcharts and architecture diagrams, requires clear module partitioning and connecting lines |
| Experimental Results Figure | Matplotlib + Seaborn | TikZ / PGFPlots | Must be generated by code to ensure reproducibility |
| Mathematical formula / theorem illustrations | TikZ | Matplotlib | Needs to integrate perfectly with LaTeX |
