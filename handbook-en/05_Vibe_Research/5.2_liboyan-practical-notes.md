**Notes for using Cursor**:

- Set the HTTP Compatibility Mode in settings to 1.1 to access the world's leading models
- Supports a rich plugin ecosystem, which can be installed as needed

### Vibe Coding Mindset Principles

- **Whatever AI can do, do not do manually**: trust AI, but review the code yourself
- **Let the goal drive the conversation**: be goal-oriented, context is king, avoid AI writing irrelevant code
- **Subtractive thinking / MVP mindset**: do not let AI produce large amounts of code in a single interaction; development should follow the Minimum Viable Product principle

### Vibe Coding Techniques

1. **Plan first, execute later**: use Plan Mode to have AI generate a plan; discuss with AI to finalize the plan before executing
2. **State explicit requirements**: including what to implement, functionality, I/O, coding conventions, etc.
3. **Small quick steps**: let AI implement the minimum functional unit and conduct Code Review and testing in time
4. **Manage context**: split complex tasks into subtasks and supplement missing context
5. **Provide debugging context**: when AI makes mistakes, provide sufficient context to help it fix the issue
6. **Become proficient with git**: use version control and retain rollback capability

## Paper Writing and Figure Drawing Techniques

### Vibe Figure Drawing Process (Four-Step Method)

**Step 1: Figure Design**
Tell the AI (e.g., through a conversation with GPT) the paper logic, and ask it to propose multiple drawing schemes for the main figure (e.g., options a to f), then select the best one.

**Step 2: Generate Prompts**
Based on the chosen drawing scheme, have GPT or Gemini generate image-generation prompts compliant with paper submission standards (e.g., using English fonts, meeting SML Oral quality requirements).

**Step 3: Generate Draft Figures**
Use the prompts with Gemini Web or Nano Banana Pro and other text-to-image models to generate figures. The generated figure is a draft; if the prompts are well-aligned with expectations, draft quality should generally be decent.

**Step 4: Draw the Vector Figure**
Use drawing tools to create a vector figure based on the draft (raster images generated by Gemini or Nano Banana cannot be used in the paper directly):
- Recommended tools: **Figma**, **OmniGraffle**, **PPT** — no hierarchy among tools; proficiency is what matters
- Icon sources: download from icon websites, or generate via GPT (you may have the AI redraw in the style of existing icons)
- PNG to vector: use tools like Edit Banana, or draw manually in Figma/PPT; SVG is the most standard vector format

### Vibe Writing Guidelines

- **Strictly prohibited**: directly copying AI-generated paper content into your own paper — you are responsible for content accuracy and the AI-generated ratio
- **Strictly prohibited**: letting AI write references or freely cite them
- **Correct usage**: provide sufficient context (research background, method, limitations, challenges, experimental data, paper figures, etc.), and let AI assist with writing ideas; but verify the accuracy of every sentence manually

## Q&A Session

**Q: How to leverage large AI models to generate feasible research ideas?** (Asked by Yuyu Luo)

Boyan Li's view: currently AI's ability to directly propose feasible ideas is limited, but one can engage in deep discussions with AI about specific problems. For example, reducing the schema cost of Text-to-SQL — by continuously "interrogating" AI's schemes and guiding it with one's own expertise, there is a high chance AI can propose feasible ideas.

**Q: Specific practices for Web icons? Can PNG be converted to SVG?**

- AI can be asked to redraw in the style of existing icons
- PNG icons can usually be placed in figures, but for large figures it is recommended to convert to SVG vector format
- PNG-to-SVG tools: Edit Banana (recommended by Prof. Luo), VI VectorRise, and other vectorization tools; can also be drawn manually in Figma or PPT
- What is downloaded from the Gemini website is PNG; PDF/SVG are vector formats

**Q: Copilot performance and model selection?**

- In Copilot you can select different models in settings; no AI tool is absolutely better than others
- Recommended to use mature IDEs or CLIs (rather than plugins), as IDEs have better context engineering
- If using the Copilot plugin, choose more powerful models such as Gemini Pro, Claude Code 4.5/4.6, etc.

**Q: Does AI have closed-loop research capability?**

Boyan Li's view: currently AI lacks a complete closed-loop research capability, but you can try distilling the paper publication cycle as context for AI to learn from, exploring the possibility of AI proposing more promising ideas.
