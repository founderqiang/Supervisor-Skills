# 2.2 How to Come Up with Ideas: "Higher, Faster, Stronger" Starting from Existing Problems

> Do not take an "axe" (solution) in hand and go looking for "wood" (a research problem).

In computer science and data intelligence, much of the time we are not opening up a completely new problem — that belongs to "Disruptive" disruptive innovation, which can only be encountered but not sought after. Rather, we are proposing improvements to an existing important problem, which belongs to "Incremental Innovation".

When you are faced with a Baseline that already has a General method, how do you systematically find an entry point (Idea)? You can draw on the Olympic spirit of "higher, faster, stronger", and, combined with the characteristics of computer systems, extend it into a five-dimensional thinking framework.

## 1. Higher: Improving Accuracy and Effectiveness

This is the most intuitive direction of improvement: existing methods are not good enough — I want to make them better.

When looking for an Idea entry point, you can consider the Information/Modality Augmentation strategy, i.e., think about whether existing methods overlook some key input or feedback signal. For example, existing Text-to-SQL LLM methods may only focus on the table schema (Schema) and the user's natural-language query. If you supplement the input with the database's specific data distribution (Data Distribution) or relevant business documents (Domain Knowledge), the model's generation accuracy usually improves substantially.

In addition, Feedback-driven Refinement is also a common line of thought in the current Agent era. For example, code generated in a single pass by an LLM often fails to run; if you introduce real error messages from the compiler or execution environment as feedback and let the Agent perform Self-Reflection and correction, you can significantly improve the final success rate.

Finally, through Error-driven Analysis — running the Baseline once, taking out all failed Cases, categorizing them, and discovering the root causes of the errors (for example, the model often fails on complex Join operations) — and then designing specialized modules in a targeted way (for example, designing a dedicated sub-Agent responsible for decomposing Join logic) is also an effective path to producing good Ideas.

## 2. Faster: Improving Efficiency and Reducing Overhead

When a method's accuracy is already high but it runs too slowly and consumes too many computational resources, "faster" becomes a highly valuable research direction.

In terms of efficiency optimization, you can consider Caching & Experience Reuse. For example, when an LLM Agent executes similar tasks, it often plans and calls tools from scratch each time. If you can cache previously successful execution trajectories (Trajectories) as an experience repository, and directly retrieve and reuse them when encountering similar tasks next time, both efficiency and Token cost will be significantly optimized.

In addition, Parallelization & Decoupling is also a classical strategy. For complex long-pipeline tasks, if a monolithic Agent executing serially is too slow, you can design a multi-agent framework, decoupling the task into multiple independent sub-tasks, letting multiple specialized Agents process them in parallel, and then merging the results.

Moreover, you can also explore Early Exit & Dynamic Routing — for simple problems, directly use a small model or a simple strategy to solve them quickly, and only invoke expensive large models or complex Agent workflows when encountering complex problems.

## 3. Stronger: Improving Robustness & Generalization

A method may perform very well on a standard dataset (Benchmark), but collapses once switched to real-world scenarios, encounters noisy data, or is tested across domains. Solving this kind of "fragility" is a very advanced Idea.

Improving robustness can start from Noise-tolerance & Fault-tolerance design. For example, users' natural-language inputs often contain ambiguity or typos. Existing methods may simply fail; if you can design an "Intent Clarification" module that lets the Agent proactively ask the user questions to eliminate ambiguity, the system's overall robustness will be better.

For unexpected failures during execution, you can design Exception Recovery mechanisms — when an API called by the Agent goes down or returns an abnormal format, it can automatically try a backup tool or adjust parameters and retry, rather than simply crashing.

To improve generalization, you can also try Decoupling — decoupling the general reasoning framework, e.g., the Agent's planning ability, from domain-specific knowledge, e.g., concrete API documentation or database Schema, thereby achieving Zero-shot cross-domain generalization.

## 4. Cheaper: Reducing Data and Annotation Cost (Data Efficiency), or the Initialization, Training, and Deployment Cost of the Method (Solution Efficiency)

In the data intelligence field, high-quality annotated data, e.g., complex SQL written by experts, high-quality Agent execution trajectories, is often extremely expensive. If your method can achieve the same results with little or even no data, this is a huge contribution.

To reduce annotation cost, you can study LLM-based Data Synthesis. Leverage powerful general large models, e.g., GPT-4o, to automatically generate large amounts of training data with pseudo-labels (Pseudo-labels), or simulate real environments to generate Agent interaction trajectories, thereby reducing dependence on human annotation.

If budget is limited, Active Learning & Human-in-the-loop is a good idea. Design an algorithm to automatically pick out the samples about which the model is most uncertain, "most worth annotating", or "most in need of human intervention", rather than blindly having humans annotate all data — this maximizes the value of human experts.

Additionally, Knowledge Distillation is also an effective way to reduce deployment cost — for example, using an expensive closed-source large model to generate high-quality reasoning processes (Reasoning Paths) and then fine-tuning a cheap open-source small model so that it reaches a comparable level on a specific task.

## 5. Broader: Cross-domain and Generality Expansion, and Vice Versa

Cleverly "transplanting" a mature and excellent idea from one field to another field that lacks such an idea can often achieve a dimensional-reduction strike.

One entry point for cross-domain innovation can be Cross-domain Transplantation. For example, introducing classic optimization ideas from the traditional database systems field, e.g., query rewriting, execution plan caching, cost estimation, into the Planning process of large-model Agents, to solve the problem of current Agents' low planning efficiency.

Another line of thought is Generalization & Unification. Existing research is often aimed at specific tasks, e.g., Text-to-SQL, Text-to-Python, data visualization, each proposing scattered methods. If a unified "Data Analysis Agent Framework" (Data Agent Framework) can be proposed that encompasses many related data-processing tasks and solves them uniformly with one elegant underlying logic, this has extremely high academic value.

## Summary Table of the Five-Dimensional Idea-Generation Framework

| Dimension | Core Goal | Common Entry-point Strategies | Typical Application Examples |
| --- | --- | --- | --- |
| Higher | Improve accuracy and effectiveness | Information/modality augmentation, feedback-driven refinement, error root-cause analysis | Introducing domain knowledge to augment Text-to-SQL inputs; leveraging compiler feedback to correct code |
| Faster | Improve efficiency and reduce overhead | Caching & experience reuse, parallelization & decoupling, dynamic routing | Retrieving and reusing Agent historical trajectories; multi-Agent parallel processing of decoupled tasks |
| Stronger | Improve robustness and generalization | Noise-tolerance & fault-tolerance design, exception recovery mechanisms, decoupled representations | Intent-clarification module to handle ambiguous inputs; automatic retry when API calls fail |
| Cheaper | Reduce data and annotation cost | Automatic data synthesis, active learning, knowledge distillation | Using large models to generate training data; extracting reasoning processes to fine-tune small models |
| Broader | Cross-domain and generality expansion | Cross-domain idea transplantation, abstraction and unified frameworks | Introducing database-optimizer ideas to improve Agent planning; proposing a general data-analysis framework |

When you feel you have "no Idea", do not just think in the abstract. Take out the current best Baseline and ask yourself the questions along these five dimensions. Any shortcoming of an existing method in some aspect is the starting point for your next top-tier paper.
