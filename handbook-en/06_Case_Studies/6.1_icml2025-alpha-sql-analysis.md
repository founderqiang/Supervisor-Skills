> **Venue**: ICML 2025

## The Thinking Model for Writing a Paper Introduction (the Introduction is a Condensed Version of the Entire Paper)

Generally speaking, the Introduction can be viewed as a "compressed version" of the entire paper: using the smallest amount of text to clearly describe the research object, thoroughly explain why the problem is difficult, and then make clear why our method is necessary. A clear writing organization is as follows: first, use a typical application scenario / running example to introduce the research background and needs; then summarize representative existing works, distilling the main limitations they reveal under key assumptions, data characteristics, workload and system constraints (usually no more than 3 points); on this basis, further characterize the essential properties and hard constraints of the problem (e.g., scale, dynamicity, heterogeneity, end-to-end overhead, correctness/consistency requirements, etc.), naturally leading to the research goal or problem definition this paper aims to address (Our Goal / Problem Formulation), or the core insight supporting the scheme design (Key Idea). Next, the key challenges to achieving this goal must be made clear (usually no more than 3 points), explaining why directly applying or simply extending existing methods is insufficient. Finally, give a methodology overview corresponding one-to-one to the challenges (overall framework and key modules), and conclude with the contribution points: problem definition/setting (if any), system/framework design, 1–2 key technical points, and sufficient experimental evaluation and analysis.

## Analyzing the Original Introduction Based on the Flowchart

### Quickly Positioning: What Type Is This Paper?

**Technique paper (new method solving an existing problem)**
- Main axis: Key Idea / Mechanism
- Goal: one sentence is enough

**In contrast, the other type of paper: Propose a New Research Problem/Setting (new problem/new setting/new task)**
- Main axis: Our Goal / Problem Formulation (the problem definition itself is a contribution)
- Key Idea: supports "why this definition is reasonable/feasible"

Alpha-SQL belongs to the former — it proposes a new technical method for an existing problem "Text-to-SQL."

The idea is to propose a fundamental improvement targeting the root cause, not a superficial patch. |

| **Methodology Overview** | **Based on this idea,** Alpha-SQL leverages a Monte Carlo Tree Search (MCTS) framework (Coulom, 2006; Xie et al., 2024) to generate and explore SQL construction actions dynamically. To facilitate efficient and effective search within the MCTS framework, we introduce the following novel techniques. **First**, to enhance reasoning capabilities during the search process, we propose the LLM-as-Action-Model, which invokes an LLM as the reasoning action model in the MCTS framework to generate step-by-step reasoning (i.e., Chain-of-Thought) after each action taken. This reasoning is stored in each node alongside the partial state, enabling Alpha-SQL to maintain context and track the LLM's thought process throughout the SQL construction process. **This ensures that each SQL construction action is both context-aware and aligned with the overall reasoning path, which can guide the search toward more promising SQL queries.** **Second**, to ensure accurate and efficient query generation during the MCTS search process, we introduce a self-supervised reward function to evaluate the quality of candidate SQL queries. Specifically, for each reasoning path, Alpha-SQL generates multiple candidate SQL queries using high-temperature sampling, filters out invalid queries, and computes a self-consistency score by comparing the execution results of the sampled queries with those of the predicted SQL. **This helps prioritize promising paths and refines the exploration process.** Finally, Alpha-SQL calculates the self-consistency scores of all candidate SQL queries and selects the SQL with the highest score as the final output. |
| **Contribution Points** | Due to page limits, this paper merges methodology with Contributions without a separate Contributions paragraph. If listed separately, Contributions would describe the overall Alpha-SQL framework, the MCTS process, LLM-as-Action-Model, and the experimental effectiveness. **In summary**, Alpha-SQL is a fine-tuning-free, plug-and-play Text-to-SQL framework that enhances small open-source LLMs for Text-to-SQL tasks. As shown in Figure 1, it can integrate and boost existing LLMs without fine-tuning on Text-to-SQL datasets. Extensive experiments show Alpha-SQL's strong performance, achieving 69.7% execution accuracy on the BIRD development set, significantly outperforming existing zero-shot methods. Ablation studies confirm the effectiveness of our reasoning actions, and performance improves with more MCTS rollouts. |
