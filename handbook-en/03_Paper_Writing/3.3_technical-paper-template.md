## A Thinking Model for Writing the Paper Introduction (The Introduction is a Condensed Version of the Entire Paper)

Generally speaking, the Introduction can be regarded as the "compressed version" of the entire paper: using the least amount of space to clearly explain the research subject, thoroughly explain why the problem is difficult, and then clearly explain why our method is necessary. A clear writing organization is: first, use a typical application scenario / running example to lead into the research background and demand; then summarize representative existing works, extracting the main limitations they expose under key assumptions, data characteristics, workloads, and system constraints (usually no more than 3 points); on this basis, further characterize the intrinsic properties and hard constraints of the problem (e.g., scale, dynamism, heterogeneity, end-to-end overhead, correctness/consistency requirements, etc.), so as to naturally derive the research goal or problem definition (Our Goal / Problem Formulation) that this paper aims to solve, or the core insight (Key Idea) that supports the solution design. Next, clearly identify the key challenges (usually no more than 3 points) for achieving this goal, explaining why directly applying or simply extending existing methods is not effective. Finally, give a method overview (overall framework and key modules) that corresponds one-to-one with the challenges, and close with the contribution points: including the problem definition/setting (if any), system/framework design, 1–2 key technical points, and thorough experimental evaluation and analysis.

### Quick Positioning: What Type of Paper Is This?

**Technique paper (new method solves an existing problem)**
- Main axis: Key Idea / Mechanism
- Goal: one sentence is enough

**Propose a New Research Problem/Setting (new problem / new setting / new task)**
- Main axis: Our Goal / Problem Formulation (the problem definition itself is the contribution)
- Key Idea: serves as support for "why defining it this way is reasonable/feasible"
